## delivery-mirror 回显循环问题

### 背景
用户报告消息重复发送，如"我收到了""已确认"等重复消息出现在 WhatsApp。

### 问题诊断
通过日志分析发现根本原因：
1. OpenClaw 使用 `delivery-mirror` 模型在消息发送后自动回显到 session 历史
2. 这些回显消息带有完整的 Conversation info metadata
3. WhatsApp 网关将这些回显消息当作用户消息再次发送
4. AI 接收到回显后，误以为是用户确认，再次回复，形成循环

### 日志证据
- 回显消息的 model 字段显示为 "delivery-mirror"
- 回显消息带有 idempotencyKey
- sender_id 为用户自己的号码（如 +8613370822913）
- 每次使用 message 工具发送后都会触发回显

### 短期解决方案
- AI 已识别此模式，不再对回显消息进行确认回复
- 忽略 sender 为用户自己号码、且内容为 AI 刚发送的回显消息

### 长期解决方案
需要联系 OpenClaw 开发团队：
- 在 WhatsApp 网关中过滤 delivery-mirror 模型的回显消息
- 或调整回显机制，不让回显被当作用户消息处理

### 相关时间
- 2026-03-26 08:11 问题诊断完成
- 2026-03-26 09:13 已成功识别并忽略多个回显消息，无循环发生
